1. Tags Form

A tag form serve para criar formulários. Você abre a tag e depois informa os campos necessários utilizando o input type, que pode ser de diversos tipos. Nesse caso, foi utilizado o type text, number e password. Por último, foi utilizado um button submit para enviar as informações preenchidas no formulário.

É possível nomear o formulário utilizando o atributo name no form para que então você possa utilizar a programação em JavaScript para refinar melhor as informações.

Podemos usar o atributo action dentro do form para enviar as informações para o servidor que desejamos.

O atributo method é usado em elementos form em HTML para especificar como os dados do formulário devem ser enviados ao servidor. Existem dois valores principais para o atributo method:

  • GET: Os dados do formulário são anexados à URL como uma string de consulta. Isso é adequado para formulários que não alteram ou adicionam dados no servidor e que têm um limite de tamanho de URL.
  • POST: Os dados do formulário são incluídos no corpo da requisição HTTP. Isso é usado para formulários que podem alterar ou adicionar dados no servidor e não têm limitações de tamanho de dados. O mais seguro é sempre utilizar o método post para que não apareçam os dados na URL.

O atributo target no elemento form do HTML é usado para especificar onde abrir a resposta do formulário após o envio. Por exemplo, você pode usar target="_blank" para abrir a resposta em uma nova aba ou _self para abrir na mesma.

O atributo autocomplete na tag form do HTML serve para controlar como o navegador preenche automaticamente os campos do formulário.

  • autocomplete="on": O navegador pode salvar e preencher automaticamente os dados do usuário em campos específicos, como nome, email e senha. Isso facilita o preenchimento de formulários repetitivos, mas pode representar um risco de segurança se o computador for compartilhado.
  • autocomplete="off": O navegador não salva ou preenche automaticamente nenhum dado do formulário. O usuário precisa digitar todas as informações manualmente, o que pode ser mais lento, mas é mais seguro.
  • Valores específicos: Você pode especificar quais tipos de dados o navegador deve ou não preencher automaticamente. Por exemplo, autocomplete="email" permite que o navegador preencha automaticamente apenas campos de email, enquanto autocomplete="off" desativa o autopreenchimento para todos os campos.

O poder do onsubmit em HTML: Domine seus formulários! O atributo onsubmit em HTML te dá a superpotência de executar ações quando um formulário for enviado. Imagine um botão de "Enviar" que, além de enviar o formulário, também valida os dados, exibe uma mensagem de sucesso ou redireciona para outra página. Tudo isso é possível com o onsubmit!

  1. O usuário clica no botão "Enviar".
  2. O evento onsubmit é acionado.
  3. O código JavaScript que você escreveu é executado.
  4. O formulário é enviado (ou não, se você cancelar o envio).

O que você pode fazer com o onsubmit:

  • Validar dados: Verifique se o usuário preencheu tudo corretamente e se os formatos dos dados estão ok (email, telefone, etc.).
  • Exibir mensagens: Mostre alertas de erro ou sucesso para o usuário.
  • Evitar envios indesejados: Cancele o envio do formulário caso a validação falhe.
  • Executar ações AJAX: Atualize partes da página sem recarregar tudo.
  • Redirecionar para outra página: Leve o usuário para um novo local após o envio.

2. Tag Input e Seus Tipos

O elemento input é a chave para criar campos de formulário interativos em HTML. Com ele, você dá vida a diversos tipos de inputs, como caixas de texto, botões, menus suspensos e muito mais!

2.1. Tipos de Input:

  • Text: O input type="text" é uma caixa de texto genérica em HTML. O ideal é sempre utilizar o label para indicação antes da caixa.

  • Number: Caixa de número genérica. Posso definir o número mínimo e máximo para esse campo utilizando o min e o max após o number; nesse caso eu defini o número mínimo como 20 e o máximo como 120. Também consigo definir se quero que ele conte de 5 em 5, 10 em 10 e assim por diante usando o step; nesse caso ele conta de 10 em 10.

  • Button: O input type="button" é um botão em HTML. O value define o que estará escrito dentro da caixa botão. Semanticamente é melhor utilizar o button para botão e não o type button, pois consigo visualizar melhor que é um botão.



  • Range: O input type="range" em HTML te dá o poder de criar sliders interativos em seus sites e aplicações! Imagine um controle deslizante para ajustar o volume, escolher uma cor ou definir o nível de zoom. Com o input type="range", tudo isso é possível e muito mais! Também é possível configurar um mínimo, máximo e o valor inicial utilizando o min, max e value respectivamente.

  • Color: No HTML, o input do tipo color permite ao usuário selecionar uma cor. Esse elemento gera um seletor de cores que, ao ser clicado, abre uma interface gráfica onde é possível escolher uma cor específica. O valor da cor selecionada é retornado no formato hexadecimal (#RRGGBB), sendo útil para personalização de interfaces, como escolher a cor de fundo, texto ou elementos gráficos em uma página web. Para criar um campo de input de cor em HTML, basta usar a tag <input type="color">.

  • Email: O input type="email" no HTML é um campo de entrada que permite aos usuários inserirem endereços de email. Ele valida automaticamente se o formato do email está correto, garantindo que o usuário forneça um endereço válido. Além disso, em dispositivos móveis, o teclado é adaptado para facilitar a digitação de emails. Isso melhora a experiência do usuário e a precisão dos dados inseridos. Para usar, basta adicionar a tag <input type="email">.

  • Date: O input type="date" em HTML permite aos usuários selecionarem uma data através de um calendário embutido. Essa funcionalidade garante a padronização do formato de data e facilita a entrada de informações precisas. Para usar, basta adicionar a tag <input type="date"> no código HTML.

  • Datetime-local: O input type="datetime-local" no HTML permite que os usuários selecionem uma data e hora local através de uma interface amigável. Isso é ideal para agendamentos e eventos com horários específicos, garantindo precisão e facilidade de uso. Para implementar, basta usar a tag <input type="datetime-local"> no código HTML.

  • Search: O input type="search" no HTML é um campo de entrada otimizado para pesquisas. Ele melhora a experiência do usuário ao oferecer funcionalidades específicas, como a possibilidade de limpar o campo com um botão embutido. Em dispositivos móveis, o teclado é adaptado para facilitar a digitação de termos de busca. Para usar, basta adicionar a tag <input type="search"> no código HTML.

  • Tel: O input type="tel" no HTML é um campo de entrada projetado para números de telefone. Ele otimiza a experiência do usuário ao adaptar o teclado para a digitação de números em dispositivos móveis, facilitando a inserção de informações precisas. Esse tipo de input não valida automaticamente o formato do número, mas garante uma interface adequada para o propósito. Para usar, basta adicionar a tag <input type="tel"> no código HTML.

  • Time: O input type="time" no HTML permite aos usuários selecionarem horários específicos com precisão, através de uma interface amigável. Ideal para agendamentos e eventos, esse campo facilita a entrada de informações de tempo, garantindo padronização e facilidade de uso. Para implementar, basta usar a tag <input type="time"> no código HTML.

  • Url: O input type="url" no HTML é um campo de entrada projetado para URLs (endereços de sites). Ele valida automaticamente se o formato do URL está correto, garantindo que os usuários forneçam endereços válidos. Além disso, em dispositivos móveis, o teclado é adaptado para facilitar a digitação de URLs. Para usar, basta adicionar a tag <input type="url"> no código HTML.

  • Week: O input type="week" no HTML permite que os usuários selecionem uma semana específica de um ano através de uma interface amigável. Essa funcionalidade é útil para aplicativos que precisam de informações precisas sobre semanas, como planejadores e agendas. Para usar, basta adicionar a tag <input type="week"> no código HTML.

  • Checkbox: O input type="checkbox" no HTML é uma caixa de seleção que permite aos usuários escolherem múltiplas opções em um formulário. Ideal para preferências e configurações, esse tipo de input oferece flexibilidade e interatividade. Para usar, basta adicionar a tag <input type="checkbox"> no código HTML.



  • Radio: O input type="radio" no HTML permite aos usuários selecionar uma única opção dentro de um conjunto de escolhas. É ideal para perguntas de múltipla escolha onde apenas uma resposta é permitida. Para usar, basta adicionar a tag <input type="radio"> no código HTML.



3. Button e Seus Tipos

A tag button é usada para criar botões em HTML. Eles podem ser personalizados e utilizados de diversas formas.

  • Button: Cria um botão simples. Pode ser utilizado com ou sem a tag <button>.

  • Submit: Cria um botão que envia um formulário.

  • Reset: Cria um botão que reseta um formulário.

4. Select e Seus Tipos

A tag select é usada para criar menus suspensos em HTML. Dentro dela, utilizamos a tag option para definir as opções.

5. Textarea

A tag textarea é utilizada para criar campos de texto maiores, onde os usuários podem inserir múltiplas linhas de texto.
