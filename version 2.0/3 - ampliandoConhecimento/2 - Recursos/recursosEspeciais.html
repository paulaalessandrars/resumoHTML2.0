2. Code, kbd e pre

2.1. Code

O code é utilizado para definir trechos de código. Ele é geralmente usado para destacar exemplos de código ou comandos dentro de um documento HTML.
  • Uso principal: Destacar código-fonte ou comandos.
  • Estilo padrão: O texto dentro de code é exibido com uma fonte monoespaçada (ou seja, cada caractere ocupa a mesma quantidade de espaço horizontal).
  • Exemplo:

    Para imprimir "Hello, World!" no console, use o seguinte comando:

    console.log("Hello, World!");

    • Neste exemplo, console.log("Hello, World!"); será exibido em uma fonte monoespaçada, destacando-o como um trecho de código.

2.2. Kbd

O kbd é usado para representar a entrada de teclado. Este elemento indica texto que o usuário deve digitar em um teclado.
  • Uso principal: Indicar teclas que devem ser pressionadas pelo usuário.
  • Estilo padrão: O texto dentro de kbd também é exibido com uma fonte monoespaçada, semelhante ao code.
  • Exemplo:

    Para copiar o texto selecionado, pressione Ctrl + C.

    • Neste exemplo, "Ctrl" e "C" são destacados como teclas que o usuário deve pressionar, geralmente em uma combinação de atalhos de teclado.

2.3. Pre

O pre é utilizado para definir texto pré-formatado. Este elemento preserva espaços em branco, quebras de linha e formatação do texto dentro dele.
  • Uso principal: Exibir texto com formatação exata, como poesia, arte ASCII, ou código com indentação específica.
  • Estilo padrão: O texto dentro de pre é exibido com uma fonte monoespaçada e a formatação original é mantida.
  • Exemplo: 
                                    function greet() {
                                    console.log("Hello, World!");
                                }
    • Neste exemplo, a função JavaScript é exibida com a indentação original preservada, tornando o código mais legível e organizado.

4. Meter e progress

Os elementos HTML meter e progress são usados para representar dados e progresso em interfaces web, mas têm propósitos distintos.

4.1. Meter

O elemento meterrepresenta uma medição escalar em um intervalo conhecido, ou uma fração de um valor. Ele é utilizado para exibir um valor dentro de uma faixa específica, como a quantidade de espaço em disco usado, o nível de bateria, ou a força de um sinal.

Atributos principais:

  • min: O valor mínimo do intervalo.
  • max: O valor máximo do intervalo.
  • value: O valor atual.
  • low e high: Definem limites onde o valor é considerado baixo ou alto.
  • optimum: Define o valor considerado ótimo.
Exemplo:

    Força do sinal Wi-Fi:

    70%
4.2. Progress

O elemento progress representa o progresso de uma tarefa, como um carregamento de arquivo ou um progresso de download. Ele é usado para indicar o andamento de uma tarefa que está em progresso.

Atributos principais:

  • max: O valor máximo do progresso (padrão é 1).
  • value: O valor atual do progresso.
Exemplo:

    Progresso do download:

    40%

6. Canvas

O elemento HTML canvas é usado para desenhar gráficos, gráficos interativos, animações, jogos e outras renderizações visuais usando JavaScript. O canvas cria uma área de desenho em HTML5 que pode ser manipulada através de um contexto de desenho (como 2D ou WebGL para gráficos 3D).

7. Novos atributos do HTML 5

HTML5 introduziu várias melhorias e mudanças em relação ao HTML4, visando melhorar a semântica, a interoperabilidade, a multimídia, e a interação com as páginas web. Aqui estão as principais diferenças: 7.1. Novos Elementos Semânticos

HTML5 introduziu novos elementos que fornecem uma estrutura mais significativa e semântica para os documentos HTML:

  • header, footer: Definem cabeçalhos e rodapés de seções ou da página.
  • article: Representa um conteúdo autônomo que pode ser distribuído independentemente.
  • section: Define seções genéricas de conteúdo.
  • nav: Representa um conjunto de links de navegação.
  • aside: Representa conteúdo tangencialmente relacionado ao conteúdo principal.
  • main: Define o conteúdo principal do documento.
  • figure, figcaption: Usados para marcar figuras e suas legendas.
7.2. Suporte Nativo para Multimídia

HTML5 inclui novos elementos para incorporar conteúdo multimídia diretamente:

  • audio: Embute conteúdo de áudio.
  • video: Embute conteúdo de vídeo.
7.3. Novos Tipos de Input e Atributos de Formulário

HTML5 introduziu novos tipos de input e atributos para formulários, melhorando a usabilidade e a validação:

  • Novos tipos de input: email, url, number, range, date, time, color, entre outros.
  • Novos atributos: placeholder, required, autofocus, pattern, min, max, step.
7.4. APIs e Funcionalidades JavaScript

HTML5 inclui várias novas APIs que permitem funcionalidades avançadas diretamente no navegador:

  • Canvas API: Para desenhar gráficos 2D e 3D.
  • Web Storage API: localStorage e sessionStorage para armazenamento local de dados.
  • Geolocation API: Para acessar a localização geográfica do usuário.
  • Web Workers: Para executar scripts em segundo plano sem bloquear a interface do usuário.
  • Web Sockets: Para comunicação bidirecional em tempo real.
  • Offline Application Cache: Permite que aplicações web funcionem offline.
7.5. Melhorias na Semântica e Acessibilidade

HTML5 promoveu uma melhor semântica e acessibilidade ao introduzir elementos mais descritivos e ao exigir uma estrutura de documento mais rigorosa. Isso facilita a interpretação do conteúdo por motores de busca e tecnologias assistivas.

7.6. Remoção de Elementos e Atributos Obsoletos

HTML5 descontinuou alguns elementos e atributos que estavam presentes no HTML4, encorajando o uso de CSS e JavaScript para estilização e comportamento:

  • Elementos removidos: font, center, big, blink, marquee.
  • Atributos removidos: align, bgcolor, border (em alguns contextos), entre outros.
7.7. Compatibilidade e Suporte para Mobilidade

HTML5 foi projetado com foco em compatibilidade com dispositivos móveis e responsividade. Ele oferece uma base sólida para o desenvolvimento de aplicações web modernas que funcionam em uma variedade de dispositivos e tamanhos de tela.

7.8. Resumo

  • Elementos Semânticos: HTML5 introduz elementos que melhoram a estrutura e a semântica do documento.
  • Multimídia: Suporte nativo para áudio e vídeo.
  • Formulários: Novos tipos de input e atributos que facilitam a criação e validação de formulários.
  • APIs JavaScript: APIs avançadas para gráficos, armazenamento, geolocalização, e mais.
  • Acessibilidade: Melhor suporte a tecnologias assistivas e melhores práticas semânticas.
  • Modernização: Remoção de elementos obsoletos e maior foco em CSS e JavaScript para estilização e comportamento.
